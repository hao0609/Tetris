# 俄羅斯方塊（Tetris）網頁版 開發計畫

## 1. 專案目標
使用純 JavaScript（不依賴框架）製作一個可在瀏覽器中遊玩的俄羅斯方塊遊戲，
支援鍵盤操作、計分、關卡加速、下一個方塊預覽，並可在桌面瀏覽器上流暢運行。

## 2. 技術選型
- **語言**：HTML5 + CSS3 + JavaScript（ES6+）
- **渲染方式**：`<canvas>` 2D Context（效能好、繪製簡單）
- **無框架、無建置工具**：單純用瀏覽器可直接開啟的靜態檔案
  - 之後若要擴充（存檔、排行榜等）可再考慮加入 build tool
- **音效（選配）**：Web Audio API 或簡單 `<audio>` 標籤

## 2.5 視覺風格：霓虹 / 賽博風
- **背景**：深黑 / 深紫藍色調，可加入細微網格線或漸層光暈
- **方塊配色**：高飽和霓虹色（如螢光藍、洋紅、黃綠、橙紅），每種方塊搭配 `box-shadow` / canvas 陰影模擬發光邊框
- **UI 文字**：等寬字體或帶科技感的字型，分數/等級使用霓虹色文字加發光效果
- **消行特效（選配）**：消除時方塊短暫閃爍或發光加強，強化回饋感
- **配色與 shape 定義**：統一放在 `constants.js`，方便日後調整風格

## 3. 檔案結構（初版）
```
tetris/
├── index.html        # 頁面結構、canvas、UI（分數/等級/下一個方塊）
├── style.css         # 版面樣式
├── main.js           # 遊戲初始化、主迴圈（game loop）
├── board.js          # 棋盤資料結構、碰撞檢測、消行邏輯
├── piece.js          # 方塊（Tetromino）定義、旋轉邏輯
├── input.js          # 鍵盤事件處理
└── constants.js       # 顏色、方塊形狀、遊戲參數等常數
```

## 4. 核心功能（MVP）
- [ ] 遊戲棋盤（預設 10 欄 x 20 列）
- [ ] 7 種標準方塊（I, O, T, S, Z, J, L），含標準顏色
- [ ] 方塊下落、左右移動、旋轉（SRS 或簡化版旋轉皆可）
- [ ] 碰撞檢測（邊界 / 已放置的方塊）
- [ ] 方塊鎖定（landing lock）與新方塊產生
- [ ] 整行消除與下移邏輯
- [ ] 計分系統（依消除行數給分，如 1/2/3/4 行分別給不同分數）
- [ ] 等級系統（分數或消行數達標後提升等級，下落速度加快）
- [ ] 遊戲結束判定（新方塊生成時立即碰撞 = Game Over）
- [ ] 下一個方塊預覽（Next piece preview）
- [ ] 暫停 / 重新開始功能
- [ ] 本地最高分紀錄（localStorage），重新整理頁面後仍保留

## 5. 進階功能（Nice to have，之後迭代）
- [ ] Hold（暫存方塊）功能
- [ ] 幽靈方塊（Ghost piece，顯示落點預覽）
- [ ] 軟降（Soft drop）與硬降（Hard drop）
- [ ] 音效與背景音樂
- [ ] 手機觸控支援（虛擬按鍵）
- [ ] 難度選擇 / 自訂棋盤大小

## 6. 操作按鍵（預設）
| 按鍵 | 功能 |
|------|------|
| ← / → | 左右移動 |
| ↓ | 軟降（加速下落） |
| ↑ | 旋轉 |
| Space | 硬降（瞬間落下） |
| P | 暫停 / 繼續 |
| R | 重新開始 |

## 7. 資料結構設計重點
- **棋盤**：使用二維陣列 `board[row][col]`，0 表示空格，非 0 表示已填色的方塊編號
- **方塊（Piece）**：儲存目前形狀矩陣（4x4）、座標 (x, y)、旋轉狀態
- **遊戲迴圈**：使用 `requestAnimationFrame` 搭配時間累計（delta time）控制下落速度，
  避免使用單純的 `setInterval` 造成不同裝置速度不一致

## 8. 開發階段規劃
1. **Day 1**：搭建 HTML/CSS 版面、canvas 初始化、繪製空棋盤
2. **Day 2**：實作方塊資料與繪製、基本下落與鍵盤移動
3. **Day 3**：碰撞檢測、鎖定機制、消行邏輯
4. **Day 4**：計分、等級、下一個方塊預覽、Game Over 畫面
5. **Day 5**：細節優化（幽靈方塊、硬降、暫停）、測試與除錯
6. **Day 6+**：進階功能與美化（視覺風格、音效等）

## 9. 確認事項（已決定）
- **視覺風格**：霓虹 / 賽博風（Neon / Cyberpunk）— 深色背景、高飽和霓虹色方塊、發光邊框效果
- **手機觸控支援**：暫不實作，列入後續迭代（第 5 節進階功能）
- **本地最高分紀錄**：需要，使用 `localStorage` 儲存最高分，重整頁面後仍保留

## 10. 里程碑驗收標準
- 可完整玩一局：方塊生成 → 移動/旋轉 → 消行 → 計分 → Game Over → 重新開始
- 遊戲速度會隨等級提升而合理加快
- 無明顯 bug（如方塊卡死、穿牆、消行誤判）
